# Code of Conduct

## Purpose

Moimio is built and maintained in the open. This Code of Conduct describes the behaviour expected from everyone participating in the project — contributors, maintainers, users asking questions, and anyone interacting in the project's spaces. The goal is straightforward: keep this a place where people can work together on the code without being mistreated.

## Expected behaviour

- Treat every person you interact with as a person of equal worth.
- Engage with disagreement honestly and charitably. Critique ideas, not people.
- Communicate clearly and in good faith. Assume the same of others until you have real reason not to.
- When you make a mistake, acknowledge it, fix what you can, and move on. When someone else makes one, prefer correction over public shaming.
- Where possible, try to resolve a conflict directly with the other person before escalating publicly.

## Unacceptable behaviour

- Harassment, intimidation, or threats of any kind, against anyone, for any reason.
- Personal attacks, insults, slander, or sustained hostility.
- Sexual advances, unsolicited sexual content or imagery, or sexualised remarks directed at any participant.
- Publishing another person's private information (real name, address, employer, contact details, photographs, etc.) without their consent.
- Sustained disruption of discussions, deliberate derailment, or trolling.
- Encouraging or assisting any of the above.

The project's spaces are not a venue for unrelated political, ideological, or religious campaigning. Disagreements about the work itself are welcome; using the project as a platform for unrelated advocacy is not.

## Scope

This Code of Conduct applies to all project spaces, including:

- The GitHub repository (issues, pull requests, discussions, code reviews, commit messages).
- Any official project communication channels we may add in the future.
- Public spaces when an individual is officially representing the project (for example, posting from a Moimio social account, presenting at a conference as a Moimio maintainer).

## Reporting

If you experience or witness unacceptable behaviour, report it to:

**`contact@moimio.app`**

All reports are read by the project maintainer and treated confidentially. Reports are handled as time allows — Moimio CE is maintained by one person in spare time; please be patient with response times. For all reporting situations (including those concerning the project maintainer), `contact@moimio.app` is the address to use.

When reporting, it helps to include:

- Where and when the incident happened.
- Who was involved (if you're comfortable saying).
- A description of what occurred.
- Any relevant links, logs, or screenshots.

You do not need to provide your real name to make a report. A pseudonymous report we can verify is better than no report at all.

## Enforcement

The project maintainer makes enforcement decisions, with the right to remove, edit, or reject contributions and to ban individuals — temporarily or permanently — for behaviour that violates this Code. Responses will be proportionate to the conduct: a private correction for a first minor lapse, a public warning for sustained or more serious behaviour, and bans where the behaviour is harmful and unrepentant.

Reports made in bad faith — fabricated incidents, weaponised reporting to silence a legitimate disagreement — are themselves a violation of this Code.
